# P2: 4択クイズ

> セットの言葉で4択クイズ。ダミー選択肢は**ベクトル近傍**で意味的に紛らわしい語を選ぶ。終了後に正誤一覧と解説。

> **注記（2026-07）**: `correctIndex` はクライアント側正誤判定の決定によりレスポンスに含める（本文表の「非送出」記載は計画当時のもの）。出題方向 `direction=wordToMeaning|meaningToWord` をユーザー指定で追加した。`scope=unanswered` は `word_meaning.isRemembered` ベースで、正解→`true` / 不正解→`false` に更新される。

- **依存**: P0（出題対象・review API・ベクトル列）, P1（埋め込み生成）
- **領域**: `server` / `webfront`
- **完了でアンロックされるもの**: P4（正誤データを忘却曲線に流す）, P6

## 目的

- 各セットの言葉で4択クイズを出題（`data-flow.md` の出題フロー）。
- 出題範囲は「全ての言葉」/「未正解の言葉」を選択可能。
- 正解はセット内からランダム、ダミー3つは正解語の `word_embedding` 近傍から取得。
- 1回のセッション終了後に正誤一覧 → 各項目タップで解説表示。

## スコープ

### やる
- クイズセッション生成 API（正解選定＋近傍ディストラクタ構築）。
- 回答送信 API（P0 の `review` を `mode='quiz'` で利用、`review_log` 記録）。
- セッション終了時の正誤一覧データ返却。
- Web: クイズ画面・回答・結果一覧・解説ビュー。

### やらない
- 出題順の忘却曲線最適化（P4。P2 ではランダム/未正解フィルタまで）。
- モバイルのクイズUI（P6）。

## データモデル差分

- 原則 P0 のスキーマ（`review_log` / `review_state` / `word_embedding`）で充足。
- 必要なら `quiz_session`（任意）を追加してセッション単位の集計を保持（MVPでは不要、Web側stateで可）。

## API 仕様

| Method | Path | 概要 |
|---|---|---|
| GET | `/quiz/:setId?scope=all\|unanswered&count=N` | N問の4択を生成して返す。各問: `{ wordId, prompt, choices[4], correctIndex(非送出) }` |
| POST | `/quiz/:setId/answer` | 1問の回答記録。body: `{ wordId, meaningId?, correct }` → `review` 経由で記録 |
| GET | `/quiz/:setId/:wordId/explain` | 結果一覧から解説（意味/例文/出典）を取得 |

- ダミー選択肢: 正解語の埋め込みで近傍検索（Limit 10）→ 同セット内から3件サンプリング。
- 埋め込み欠損語（P1未補完）はテキスト類似 or ランダムでフォールバック。

## 主要UI / 画面

- クイズ開始（範囲選択: 全て / 未正解）。
- 出題画面（4択、即時正誤フィードバック）。
- 結果一覧（各問の正誤）→ タップで解説。

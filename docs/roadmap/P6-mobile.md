# P6: モバイル移植（Expo）

> P1〜P5 で**確定した仕様**を Expo アプリに移植し、Web と同じ API を叩くモバイル体験を整える。コンセプト「日常で言葉を集める」の主戦場。

- **依存**: P1〜P5（仕様確定後に着手）
- **領域**: `mobile` / `uiux`
- **完了でアンロックされるもの**: P7
- **注記**: 下流フェーズのため、本ファイルの仕様は Web 実装確定に合わせて随時更新する（着手時点で精緻化）。

## 目的

- 現状 Expo 初期シェルのみのモバイルアプリに、v1 コア機能の画面を実装。
- Web と同一の API（Hono）を共有し、ロジック重複を避ける。
- モバイルならではの**カメラ/位置情報での言葉集め**を活かす。

## スコープ

### やる
- 認証（Better Auth、Expo Secure Store でのセッション保持）。
- 単語/単語セットの一覧・登録・編集（画像はカメラ/ライブラリ、署名URLで R2 直PUT）。
- AI補完の補完中→完了表示（P1 と同等挙動）。
- クイズ（4択）／フラッシュカード（スワイプ）／間隔反復の出題。
- ダッシュボード（word cascade 等の演出は可能な範囲で）。
- 課金導線（基本は Web 決済へ誘導、ストア審査要件を確認の上で方式決定）。

### やらない
- v1 範囲外の演出・通知（「一年前の今日」通知などは v1 後）。

## データモデル差分

- なし（既存 API を消費）。モバイル固有の永続化は Secure Store のみ。

## API 仕様

- 新規エンドポイントは原則なし。P0〜P5 で定義した API を利用。
- 画像は `data-flow.md` の署名付きPUT URL フローでモバイルから直接 R2 へ。

## 主要UI / 画面

- ログイン / セット選択 / 一覧 / 登録ドロワー（カメラ・位置）/ 編集。
- クイズ画面・結果一覧 / カード（スワイプ）/ 閲覧・ダッシュボード。
- ナビゲーション構成（タブ等）。

## タスク（Issue 化）

1. ナビゲーション基盤と認証（Secure Store セッション）
2. API クライアント整備（Web と共有可能なら `packages` へ切り出し検討）
3. セット選択＋単語一覧画面
4. 単語登録（カメラ/位置/画像直PUT）＋AI補完中表示
5. 単語編集画面（Notion-like＋チャット send）
6. クイズ画面＋結果一覧
7. フラッシュカード（スワイプ）画面
8. ダッシュボード画面
9. 課金導線（Web 誘導 or ストア課金の方式決定と実装）

## 完了条件

- モバイルから登録〜AI補完〜クイズ/カード学習〜ダッシュボードまで一通り動く。
- Web と同じ API・データで整合する。
- 実機（iOS/Android いずれか）で主要フローが動作確認済み。
